## 项目说明

插件会把本地书签导出为快照 JSON，并读写到仓库指定路径（默认 `bookmarks/snapshot.json`）。

核心动作：

- `推送本地到云端`：将本地书签写入远端仓库
- `拉取云端覆盖本地`：将远端快照导入本地并覆盖
- `导出书签 JSON`：离线备份
- `导入书签 JSON`：离线恢复

---

## 功能总览

已支持：

- GitHub / Gitee 双平台
- 两个平台可分别配置，并在弹窗按需选择推送/拉取来源
- 推送目标支持：
- 当前平台
- 同时推送到所有已配置平台
- 自动定时推送（最小 15 分钟）
- 冲突检测、差异预览、手动确认后强制同步
- 设置页支持“配置导出 / 配置导入”（包含 Token 与自动同步参数）
- 弹窗显示本地书签数量，以及 GitHub / Gitee 各自的云端书签数量

边界说明：

- 自动同步只做“推送”，不会自动拉取
- 拉取必须选择单个平台执行（不会同时从多个平台拉取）
- 冲突时自动任务会跳过，不会自动强制覆盖

## 安装插件

### Chrome

1. 打开 `chrome://extensions`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目根目录（含 `manifest.json`）

### Edge

1. 打开 `edge://extensions`
2. 开启“开发人员模式”
3. 点击“加载解压缩的扩展”
4. 选择本项目根目录（含 `manifest.json`）

发布打包注意：

- 商店上传包必须包含 `_locales` 目录（如 `_locales/zh_CN/messages.json`）
- 若 `manifest.json` 中存在 `default_locale` 但压缩包缺少 `_locales`，会触发包校验错误

## 配置说明（设置页）

点击弹窗右上角齿轮进入设置页。

设置页负责“仓库连接参数”和“自动同步参数”，不负责执行推送/拉取动作。

| 字段 | 说明 | 示例 |
|---|---|---|
| 同步提供方 | 当前编辑哪一套平台配置（GitHub 或 Gitee） | `GitHub` |
| Token | 平台访问令牌 | `ghp_xxx` |
| 用户/组织 | owner 名称 | `your-name` |
| 仓库名 | repo 名称 | `bookmark-sync` |
| 分支 | 目标分支 | `main` / `master` |
| 文件路径 | 快照文件路径 | `bookmarks/snapshot.json` |
| 启用自动推送 | 是否开启定时推送 | 开 / 关 |
| 自动推送间隔(分钟) | 最小 15 分钟 | `60` |

说明：

- GitHub 与 Gitee 的配置会分别保存
- `自动推送间隔` 输入低于 `15` 时会自动校正为 `15`
- 设置页会显示提示文案：最小 15 分钟，低于该值自动调整
- 可在设置页导出配置 JSON，并在新设备导入，避免重复输入 Token/仓库信息

### 配置导出/导入

- 入口位置：设置页 `备份与迁移` 分组
- `导出配置 JSON`：导出当前全部平台配置（含 Token）与自动同步设置
- `导入配置 JSON`：导入后会立即保存并回填到设置页
- 安全提示：导出文件包含敏感令牌，请仅在可信环境保存与传输
- 说明：`备份与迁移` 分组同时包含“书签数据备份”和“配置参数备份”

## 弹窗同步说明

弹窗顶部右上角齿轮用于打开设置页；同步操作在弹窗内执行。

说明：书签备份与配置迁移统一在设置页完成，不再放在弹窗。

### 数量展示

- 弹窗会实时显示：
- 本地书签数量
- GitHub 云端书签数量
- Gitee 云端书签数量

### 拉取平台

- 下拉框只显示“已完成配置”的平台
- 拉取时会从选中的平台读取并覆盖本地
- 拉取不支持“同时多平台”

### 推送平台

- 下拉框只显示“已完成配置”的平台
- 当推送目标是“当前平台”时，推送到所选平台

### 推送目标

- `当前平台`：按“推送平台”下拉框的选中项推送
- `全部已配置平台`：会并行推送到所有已配置平台

注意：

- 当选择“全部已配置平台”时，`推送平台` 下拉会隐藏
- 该推送目标会被保存，并用于后续自动同步

## 自动同步说明

自动同步由设置页开关控制。

行为规则：

- 自动任务仅执行“推送”
- 自动任务的推送目标取决于弹窗中保存的“推送目标”：
- 当前平台：按当前配置平台推送
- 全部已配置平台：向所有已配置平台推送
- 自动间隔最小值为 15 分钟

---

## 首次同步建议

首次请先确定“以哪一边为准”：

- 以本地为准：执行 `推送本地到云端`
- 以远端为准：执行 `拉取云端覆盖本地`

如果远端还没有快照文件，请先推送一次创建快照。

## 冲突处理

检测到冲突时会显示差异预览（本地/远端数量及样例）并弹出确认。

你可以：

1. 取消本次操作
2. 选择强制继续（覆盖另一侧）

建议先执行一次 JSON 导出备份，再做强制同步。

## 备份与恢复

- 入口：设置页 `备份与迁移`
- 备份：`导出书签 JSON`
- 恢复：`导入书签 JSON`

这是最直接的兜底手段，建议在大改动前先导出。

## GitHub 配置流程

### 1. 创建仓库

1. 在 GitHub 新建仓库（建议 `Private`）
2. 记录 owner、repo、默认分支（通常 `main`）

### 2. 创建 Token

路径：
`Settings -> Developer settings -> Personal access tokens -> Fine-grained tokens`

关键权限：

1. `Repository access` 选 `Only select repositories`
2. 选择目标仓库
3. `Contents` 权限设为 `Read and write`
4. 生成后复制 Token（仅显示一次）

### 3. 在设置页填写并测试

1. 打开扩展弹窗，点击右上角齿轮
2. 选择 `GitHub`
3. 填写 Token / owner / repo / branch / path
4. 点击 `保存设置`
5. 点击 `测试连接`

---

## Gitee 配置流程

### 1. 创建仓库

1. 在 Gitee 新建仓库（建议私有）
2. 记录 owner、repo、默认分支（常见 `master` 或 `main`）

### 2. 创建私人令牌

在 Gitee 个人设置中创建“私人令牌（PAT）”，确保具备仓库内容读写权限。

### 3. 在设置页填写并测试

1. 打开扩展弹窗，点击右上角齿轮
2. 选择 `Gitee`
3. 填写 Token / owner / repo / branch / path
4. 点击 `保存设置`
5. 点击 `测试连接`

## 常见问题

| 现象 | 原因 | 处理方式 |
|---|---|---|
| 弹窗下拉看不到平台 | 对应平台配置未填完整 | 先到设置页补齐 Token/owner/repo/branch/path 并保存 |
| 推送平台下拉消失 | 选择了“全部已配置平台” | 这是正常行为，改回“当前平台”即可显示 |
| 输入 11 分钟后变成 15 | 最小间隔限制生效 | 正常行为，系统会自动调整并提示 |
| 远端文件不存在 | 先拉取但远端尚无快照 | 先执行一次推送 |
| 401 / 403 | Token 无效或权限不足 | 重新生成 Token 并确认仓库读写权限 |
| 冲突频繁 | 多设备并发改动 | 采用“先拉取再修改，改完立即推送” |
| 卸载扩展后重装，配置不见了 | 扩展使用 `chrome.storage.local` 保存配置 | 卸载通常会清空该存储，重装前请导出备份并保存配置参数 |

---

## 安全建议

1. 使用私有仓库
2. Token 仅授予必要权限
3. 不在公开场景泄露 Token
4. 定期轮换 Token
5. 强制覆盖前先做 JSON 备份
